◆ setRequestedKeepAliveTime()

void com.softing.opc.client.api.DaSubscription.setRequestedKeepAliveTime ( int value)
inline

Sets the requested maximum period between updates sent to the client. Clients can set the keep-alive time for a subscription to cause the server to provide client callbacks on the subscription when there are no new events to report. Clients can then be assured of the health of the server and subscription without resorting to pinging the server with calls to GetStatus. Using this facility, a client can expect a callback (data or keep-alive) within the specified keep-alive.
